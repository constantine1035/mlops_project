# Wine Quality Classification Service

**Проблема.**  
Производители вина хотят заранее понимать, насколько высокое качество будет у будущего напитка.  
Сенсорная оценка занимает много времени и дорогостоящих дегустаторов. Цель сервиса – автоматизировать предиктивную оценку.

**Решение.**  
Сервис получает значения 13 химических характеристик вина из классического *Wine Dataset* библиотеки scikit‑learn и выдаёт бинарный прогноз: 1 – образец относится к третьему классу, 0 – иначе.  
Модель логистической регрессии обучена на 178 записях исходного набора. Модуль переобучения запускается ежедневно scheduler’ом и обновляет файл модели.  
Инференс‑сервис автоматически подхватывает новую модель без перезагрузки.

| Компонент | Технологии | Назначение |
|---|---|---|
| ML‑сервис | Python, FastAPI, Scikit‑Learn, Prometheus | Выполняет предсказания, сохраняет результаты в PostgreSQL, публикует сообщения в Kafka, экспортирует метрики |
| UI | Streamlit | Позволяет пользователю ввести параметры вина, получить прогноз и посмотреть историю предсказаний |
| База данных | PostgreSQL | Хранит историю запросов для аудита и анализа |
| Scheduler | Apache Airflow | Раз в сутки запускает переобучение модели с использованием скрипта `train.py` |
| Message Broker | Kafka | Асинхронно отправляет сообщения с результатами предсказаний (можно интегрировать downstream‑сервисы) |
| Мониторинг | Prometheus + Grafana | Отслеживает количество запросов к сервису и время отклика |

Количество запросов к эндпойнту `/predict` мониторится в Grafana (панель *Prediction Requests per Second*).  
История всех предсказаний доступна в UI.

При локальном запуске сервис обрабатывает более 200 запросов/с. Bottleneck – база данных. Возможна масштабируемость через горизонтальное масштабирование и кеширование.
